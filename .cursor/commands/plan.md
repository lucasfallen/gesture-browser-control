# Planejar Funcionalidade (Kiro Style)

## Descrição
Gera um plano de desenvolvimento estruturado (Requisitos, Design e Tarefas) para uma nova funcionalidade ou tarefa, seguindo a metodologia orientada por especificações (Spec-Driven Development).

## Instruções
1.  Leia o prompt do usuário sobre a funcionalidade desejada.
2.  Crie uma nova pasta em `.specs/[nome-da-feature]/`.
3.  Nesta pasta, crie três arquivos Markdown:
    *   `requirements.md`: Descreva os requisitos e critérios de aceitação usando a notação EARS (Easy Approach to Requirements Syntax).
    *   `design.md`: Proponha a arquitetura, design de sistema e tecnologias a serem usadas.
    *   `tasks.md`: Crie uma lista de tarefas discretas e sequenciadas que o agente pode executar para implementar o design.
4.  Após a criação dos arquivos, informe ao usuário que o plano está pronto para revisão e aprovação antes da implementação.
