## Instruções (Revisadas para garantir atualização de tasks.md)
1. Analise apenas a próxima tarefa a ser feita (marcada como PENDING ou TODO) no `@.specs/[pasta-da-feature]/tasks.md`.
2. EXPLIQUE o plano de ação detalhado para esta ÚNICA tarefa, citando as regras que irá seguir de `.cursor/rules/`.
3. SOLICITE APROVAÇÃO do usuário explicitamente antes de tocar no código.
4. Após a aprovação, execute a tarefa.
5. CRÍTICO: Marque a tarefa como CONCLUÍDA no arquivo `tasks.md`.
6. **SOLICITE A PRÓXIMA TAREFA:** Pergunte ao usuário se ele deseja aprovar o plano para a próxima tarefa listada no `tasks.md` até que todas sejam concluídas.
